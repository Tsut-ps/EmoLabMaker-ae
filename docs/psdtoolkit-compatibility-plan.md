# PSDToolKit 互換化 計画

EmoLabMaker を PSDToolKit(oov 氏 / AviUtl 用)向けに作られた立ち絵 PSD と
そのワークフローに対応させるための調査結果と実装計画。

## 背景と目的

ニコニコ動画等で配布される立ち絵 PSD の多くは PSDToolKit の規約で作られている。
これを After Effects + EmoLabMaker でそのまま活用できるようにする。

### PSDToolKit の仕様(調査結果)

- **レイヤー命名規則**
  - `*` prefix: 兄弟レイヤー間で排他表示(ラジオボタン)
  - `!` prefix: 強制表示(常に表示、非表示にできない)
  - `:flipx` / `:flipy` suffix: 左右/上下反転バリエーション
- **口パク あいうえお@PSD**: lab ファイルの母音タイミングで あ/い/う/え/お/ん の6口形状を切替。子音は基本「ん(閉じ)」扱い
- **目パチ@PSD**: 間隔・速度パラメータで自動まばたき

参考: [PSDTool マニュアル](https://oov.github.io/psdtool/manual.html) /
[PSD アニメーション効果](https://oov.github.io/aviutl_psdtoolkit/psd.html) /
[準備オブジェクト](https://oov.github.io/aviutl_psdtoolkit/prep.html)

### 実装形態の判断: 単一 .jsx を継続

「スクリプトに留めるべきか」を調査した結論:

- CEP は CEP 12(AE 25.0 同梱)が最後のメジャーバージョンで、以後はセキュリティ修正のみ
- UXP の After Effects 対応は未リリース(時期未定)
- AE は PSD をネイティブインポートできる(レイヤー名・グループ構造・表示状態を保持)ため、
  CEP + Node.js による PSD バイナリ解析は不要
- 単一ファイル配布の手軽さは動画制作者というユーザー層に合っている

→ **今パネル(CEP)化する利点は薄い。単一 .jsx のまま続行し、UXP の AE 対応がリリースされたら再検討する。**

## 実現する機能

1. **PSD 素材互換**: AE 標準機能で読み込み済みの PSD コンポを解析し、`*`/`!` 命名規則を解釈して表情切替を自動セットアップ。再実行で既存レイヤーに更新をかけられる(冪等)
2. **目パチ**: 自動まばたきエクスプレッション生成。**開き目表情のときだけ**まばたきし、それ以外は表情切替に従う
3. **口パク改善**: 音素→あいうえお口形状の割り当てを UI で行えるようにし、レイヤーごとの手動リネームを不要にする。**しゃべっている間だけ口を上書きし、それ以外は表情切替に従う**

## 主要な設計判断

| 論点 | 判断 |
|---|---|
| UI 構成 | タブ3「PSDセットアップ」を新設(読み込み済み PSD コンポの自動セットアップ + 目パチ)。口形マッピングはタブ2「口パク」内に新パネル追加 |
| PSD の読み込み | **スクリプトは `importFile()` を実行しない**。バグ防止のため読み込みは AE 標準のインポート(コンポジションとして読み込み)に任せ、スクリプトは読み込み済みコンポの解析と既存レイヤーへの更新のみを行う |
| 再実行(更新) | セットアップは**冪等**に設計。再実行時は既存の制御レイヤー・登録・リネームを壊さず、差分(新規レイヤーの登録、エクスプレッションの再設定)だけを適用する |
| **機能間の競合** | 目パチ・口パクの対象(閉じ目・口形状)は `*` 付きレイヤーとして表情切替にも登録される同一レイヤーである。**奪い合いを避けるため、目パチ・口パクは「自分がアクティブな間だけ上書きし、それ以外は表情マーカーのロジックにフォールバックする」合成エクスプレッション**にする(emo 単独式の置き換えとして適用し、emo シグネチャも合成式に含めるため、レイヤー選択タブからは登録済みのまま見える) |
| 音素→口形状 | **エクスプレッション埋め込み方式**(`var myPhonemes = ",a,e,";` を焼き込み)。レイヤーをリネームしないので PSD 由来の可読名と emo2layer 登録を保全。emo 未登録レイヤーに適用した場合のフォールバックは「閉じ口」(どの母音リストにも属さない音素=子音 or 音素なしで表示)とし、子音の列挙は不要 |
| コンポ名の一意化 | PSD グループは「目」「口」など汎用名のコンポになり、**複数キャラ運用で同名コンポが衝突**する(エクスプレッションは `comp("名前")` でグローバル参照するため不定動作になる)。セットアップ時にグループコンポを `<ルートコンポ名>_<グループ名>` へリネームして一意化する(前置済みならスキップ=冪等) |
| `:flipx` 等 | v1 ではスキップ + 結果レポートに警告のみ(AE では Scale 反転で代替可能)。スキップ時も enabled は PSD 由来の表示状態を維持する |
| 後方互換 | 既存の `[Emo]`/`[Lab]`/`[P]` データ・既存エクスプレッションは一切変更しない。すべて y バンプ(機能追加)で収める |
| シグネチャ | 既存 `emo2layerCtrlMarker` を PSD 自動登録でも再利用。新設: `emoBlinkAuto`(目パチ)、`lab2layerPhonemeMap`(口形マッピング)。合成式は該当する複数シグネチャを併記し、解除時は emo 単独式に復元する |

## 実装ステップ

対象: `EmoLabMaker.jsx`(全実装)、`README.md`(ドキュメント)

### Step 1: 共通基盤(リファクタ、挙動不変)

- `registerSelectedLayers` から汎用版 `registerLayers(targetComp, ctrlCompName, layers, undoName)` を抽出し、既存関数は薄いラッパー化(自動セットアップが選択状態に依存せず登録できるように)
- `writeMarkerName` から `writeMarkerNameAtTime(ctrlComp, targetCompName, time, name)` を抽出(時刻 0 にデフォルト表情マーカーを書くため)
- `unregisterSelectedLayers` のパターンを一般化した `removeExpressionBySignature(layers, signature)` を追加
- **`buildOpacityExpression` をコア部品化**: マーカー名取得部(`findCtrlLayer` + `getCurrentMarkerName`)を `buildEmoMarkerSnippet(ctrlCompName, targetCompName)` として切り出し、emo 単独式・目パチ合成式・口パク合成式の3者で共有する。レイヤーの既存エクスプレッションから ctrl コンポ名/対象コンポ名を読み取るヘルパー `parseEmoContext(layer)` も用意(合成式適用時に emo 登録を引き継ぐため)
- 定数追加: `BLINK_SIGNATURE`, `LAB_MAP_SIGNATURE`

### Step 2: PSD 命名規則パーサと構造走査

- `parsePsdLayerName(name)` → `{ base, exclusive(*), forced(!), flipx, flipy }`(`*!` 複合は順不同で剥がす)
- `scanPsdCompTree(rootComp)`: AE の PSD インポート(COMP)では PSD グループ=ネストコンポになるため、コンポごとに `{ exclusiveLayers, forcedLayers, flipSkipped, defaultLayer }` を再帰収集。**ルートコンポ直下の `*` レイヤーも走査対象にする**(グループなしのフラット PSD 対応)。`defaultLayer` は exclusive のうち `layer.enabled === true` のもの(PSD の表示状態を継承)、なければ先頭
- **`*` レイヤーが 1 枚もないグループ(線画・塗りなど)は制御レイヤーを作らずスキップ**する
- 同一コンポ内の重複レイヤー名は ` (2)` 付与でリネーム(`dedupeLayerNames`)。`*` prefix は登録前に剥がしてリネーム(マーカー名・グリッドボタンに `*` を残さない)

### Step 3: 読み込み済み PSD の自動セットアップ / 更新(タブ3)

- PSD の読み込み自体は AE 標準のインポートをユーザーが行う前提。タブ3にはその手順の案内文を表示し、スクリプトは読み込みを行わない
- **対応する読み込み方式**: 「コンポジション」「コンポジション - レイヤーサイズを維持」のどちらでも動作する(セットアップはレイヤー名・コンポ構造・表示状態・不透明度のみに依存し、レイヤーサイズやアンカー位置に依存しないため)。案内文では「レイヤーサイズを維持」を推奨(後からパーツを回転・スケールする際にアンカーがパーツ中心で扱いやすい)。「フッテージ」読み込みはコンポが生成されないため対象外(ドロップダウンに現れず自然に弾かれる)
- UI: ルートコンポのドロップダウン(プロジェクト内コンポ一覧、既存 `rebuildDropdown` を再利用)+ 制御コンポのドロップダウン + 「解析」ボタン
- **解析結果の確認ダイアログ**: 検出したグループ一覧(グループ名 / 排他レイヤー数 / デフォルト表情)をチェックリストで提示し、不要なグループを**除外**してから「セットアップ / 更新」を実行できるようにする。誤検出の確認画面を兼ねる
- `autoSetupPsd(rootComp, ctrlComp, includedGroups)`: グループごとに **コンポ名を `<ルート名>_<グループ名>` へ一意化リネーム** → dedupe → `createCtrlLayer`(既存があればそれを返す仕様なので再実行安全)→ `registerLayers` → `!` レイヤーは `enabled = true` のみ(未登録)→ flip はスキップ記録。全体を 1 つの undo group に
- **冪等性ルール(再実行 = 既存レイヤーへの更新)**:
  - 登録済みレイヤー(`isRegistered`)もエクスプレッションを再設定して最新の生成内容に更新する。**目パチ・口パクの合成式が設定済みのレイヤーは合成部分を保持したまま emo 部分のみ更新**する(シグネチャで判別)
  - コンポ名の一意化・`*` 剥がしリネームは、適用済みの名前には何もしない
  - **時刻 0 のデフォルト表情マーカーは、制御レイヤーにマーカーが 1 つもないときだけ書き込む**(既存エクスプレッションはマーカーなし時 opacity 0 のため初回は必須。ユーザーが打ったマーカーは上書きしない)
  - AE 内で手動追加されたレイヤーは差分として登録する
- 完了後レポート(グループ数 / 新規登録数 / 更新数 / リネーム一覧 / flip スキップ一覧)。レイヤー選択タブの一覧を再構築し、即使える状態にする

### Step 4: 目パチ(タブ3内パネル)

- `buildBlinkExpression(params, role, emoContext)`: **emo マーカーのロジックと合成した式**を生成する。
  挙動: **現在の表情マーカーが「開き目」に割り当てたレイヤー名のいずれかであるときだけ**、time ベースのまばたきスケジュールが発動し、位相に応じて 閉じ目/中間目 を上書き表示する。それ以外の表情(笑い目・ジト目など)のときは emo マーカーのロジックそのままに従う(=不自然なまばたきをしない)
  - パラメータ: 平均間隔(既定 4.0 秒)/ 閉じ速度 / 閉じ維持 / ランダム率。開き目レイヤー名の CSV を式に埋め込む(複数の開き目を「まばたく表情」として割当可能)
  - `seedRandom(サイクル番号, true)` で決定論化し、開き/中間/閉じの全レイヤーが同一スケジュールを共有(レイヤー間同期、マーカー不要)
  - 中間レイヤー未指定時は mid 区間を closed に畳む
- 適用対象は目グループの全レイヤー(開き/中間/閉じ + その他の表情目)。emo 登録済みなら `parseEmoContext` で制御情報を引き継いで合成式に置き換える。emo 未登録レイヤーのみの適用も可(その場合は単独の time ベース式)
- 「解除」は合成式から emo 単独式へ復元(emo 未登録だった場合は式を除去して不透明度 100)
- UI: 開き / 中間(任意)/ 閉じ目の割当ボタン + 間隔・速度・ランダム% 入力 + 「目パチ設定」「解除」

### Step 5: 音素→口形状マッピング(タブ2拡張)

- 既存のインラインエクスプレッションを `buildLabNameMatchExpression()` に抽出(挙動不変)
- 新規 `buildLabMappedExpression(targetCompName, phonemeCsv, emoContext)`: **現在時刻に有効な `[P]` 音素があるときだけ**口形状を上書きし(自分の `myPhonemes` に一致で 100)、音素がないときは emo マーカーのロジックにフォールバックする合成式(=しゃべっていない間は表情側の口が生きる)。emo 未登録レイヤーに適用した場合のフォールバックは「閉じ口」レイヤーのみ 100
  - 閉じ口レイヤーには全シェイプ統合 CSV も埋め込み、「どの母音リストにも属さない音素(=子音)→ 閉じ」を子音列挙なしで実現
- プリセット(PSDToolKit 互換): `あ: a / い: i / う: u,w / え: e / お: o / ん(閉じ): N,cl,Q,pau,sil,br + 未割当フォールバック`。永続化はしない(エクスプレッションに焼き込み、再適用で更新)
- UI: 6行(あ/い/う/え/お/ん)× { 音素 CSV 入力 + 選択レイヤー割当ボタン + 割当表示 } + 「自動割当」(レイヤー名に あ/い/う/え/お/ん/閉 を含めばヒューリスティック割当)+ 「マッピング適用」「解除」(解除は emo 単独式へ復元)
- 既存の名前マッチ版「口パク設定」はそのまま残す(後方互換)

### Step 6: 仕上げ

- ヘルプダイアログとタブ3説明、README に新タブの使い方・PSDToolKit 命名規則対応表(`*`=排他→自動登録 / `!`=強制表示 / `:flipx`=v1 未対応)・口形マッピング手順・既知の制限・バージョン履歴を追記。ヘッダの `@version` 更新

## 既知の制限(README に明記する)

- **PSD 側の構造変更(レイヤー追加など)は AE の既存コンポに反映されない**。AE のフッテージ再読み込みはピクセルのみ更新する。構造が変わったら再インポート(=新しいコンポ群)+ 再セットアップが必要で、打ったマーカーは旧制御レイヤーに残る。「更新」が効くのは AE 内で手動追加したレイヤーや設定の再適用まで
- **セットアップ後にコンポを改名しない**こと。エクスプレッションの `comp("名前")` 参照と `[Emo] 名前` の対応が切れる(「更新」で式は再生成できるがマーカーは旧制御レイヤーに残る)
- 表情切替はグループ(目・眉・口など)ごとに対象コンポを選んでマーカーを打つ方式。複数グループ一括の「表情セット切替」(PSDTool のお気に入り相当)は v2 で検討

## 検証方法

1. エクスプレッション生成関数の出力を目視・構文検証(ExtendScript は ES3 準拠 — `const`/`let`/アロー関数は不可)
2. `parsePsdLayerName` / lab パース等の純粋ロジックは簡易テストで検証
3. AE 実機で一連フローを確認: PSDToolKit 向け配布立ち絵 PSD を AE 標準インポート(コンポジション)で読み込み → 自動セットアップ → 表情グリッドから切替 → lab 読込 → 口形マッピング適用 → 目パチ設定 → **再度「自動セットアップ / 更新」を実行して既存セットアップが壊れない(冪等)こと**
4. **機能間の競合検証**: 自動セットアップ済みの目・口グループに目パチ / 口形マッピングを適用 → 表情切替・目パチ・口パクが共存すること(笑い目中にまばたかない、無音時に表情の口が出る)、解除で emo 単独式に戻ること
5. **複数キャラ検証**: 2 つの PSD をセットアップし、コンポ名一意化により表情切替が互いに干渉しないこと
6. 後方互換: 既存プロジェクトの [Emo]/[P] レイヤーが新版パネルで認識・操作できること
