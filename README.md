# EmoLabMaker

<img width="1280" height="720" alt="TV - 1" src="https://github.com/user-attachments/assets/7835d4a9-a872-4983-a8dc-c6820a1ea297" />

立ち絵の表情と口パクをかんたんに切り替えられる Ae 用スクリプト  

## これ is 何？

EmoLabMaker は、

- 立ち絵の階層表示で表情切り替え
- lab ファイルを使った口パク、自動目パチ
- PSD 形式（PSDToolKit 互換対応）や任意のレイヤー構造を立ち絵として自動セットアップ

をかんたんに行うことができる、After Effects のスクリプトです  

After Effects 上でソフトウェアトークの動画編集を作りたい方、立ち絵 MV やカットシーンを作りたい人向けです

https://github.com/user-attachments/assets/7843ca7f-e059-45b1-8d44-26be9f45a0a5

（立ち絵: miwashiba / ボイス: 鳴花ヒメ・ミコト）

もともとあったスクリプト 2 つ `emo2layer`（表情切り替え）と `lab2layer`（口パク機能）を統合したスクリプトで、  
「セットアップ / 立ち絵 / 口パク / 目パチ」の 4 つのタブを切り替えて使います

> [!NOTE]
> v2 から大幅に刷新し、PSDToolKit 互換対応により、機能を立ち絵タブに統合しました  
> 個別セットアップ機能は廃止し、まとめてセットアップできるようにしています  
> 口パクのレイヤー名を変更する方式も廃止し、割り当て方式に一本化したことで、より自由度が向上しています

> [!WARNING]
> 現在 β 版です。ある程度バグを取り除いているのでとりあえず動くと思います（無保証）  
> 私が使えればOKみたいな作りをしています  
> 不具合や改善できる場所があれば、[Issues](https://github.com/Tsut-ps/EmoLabMaker-ae/issues) か [マシュマロ](https://marshmallow-qa.com/tsut_ps) に書いてもらえると、とっても助かります

> [!IMPORTANT]
> x.y.z の x のバージョンが更新されたら破壊的変更（それまでのデータとは互換性がない）です。yは機能追加、zはバグ修正


## インストール方法

1. [**Releases**](https://github.com/Tsut-ps/EmoLabMaker-ae/releases/) から `EmoLabMaker.jsx` をダウンロード
2. 以下のいずれかの場所に配置:
   - **Windows**: `C:\Program Files\Adobe\Adobe After Effects <version>\Support Files\Scripts\ScriptUI Panels\`
   - **macOS**: `/Applications/Adobe After Effects <version>/Scripts/ScriptUI Panels/`
3. After Effects を再起動
4. メニューから `ウィンドウ` → `EmoLabMaker.jsx` を選択

> [!NOTE]
> mac 環境での動作チェックはしていないので、動かなかったら自由に改変してください  
> ある程度、スクリプトの高さと幅が必要です（使うときはワークスペース機能を活用すると便利です）


## 開発・ビルド

本体ソースは保守性のためファイルを分割しています   
単純に関数をまとめて書いて、`build.js` で、1 枚の `dist/EmoLabMaker.jsx` に連結して配布としています  
ディレクトリ構成・ビルド・リリース手順は **[DEVELOPMENT.md](DEVELOPMENT.md)** を参照してください。

```sh
node build.js   # src/*.jsx -> dist/EmoLabMaker.jsx を生成
```

## 1. 立ち絵の事前準備を行う「セットアップ」タブ

<img width="432" height="174" alt="タイトルなし" src="https://github.com/user-attachments/assets/fe0f3b17-1a94-4b82-8739-6e318eb18ade" />

画像レイヤー構造を読み込んで、表情や衣服の切替ができるように自動セットアップします  
PSDToolKit（AviUtl 用）向けに作られた立ち絵も対応しています

### 使い方

1. **After Effects 標準の読み込みで立ち絵を追加**  
   PSD 形式の場合は、「コンポジション - レイヤーサイズを維持」推奨  
2. 「セットアップ」タブの〈ルート〉で、読み込んだ立ち絵を選択
3. 〈制御〉にマーカーを書き込む場所を選択
   - 立ち絵と同じ → まとまってみやすい
   - メインシーンとするコンポ → 表情調整がしやすい（推奨: シャイモード活用）
5. `解析してセットアップ / 更新` を押す
6. 確認ダイアログで、実際に使うもの（目 / 口 / 眉 など）を選択して実行

セットアップ後は、「立ち絵」タブの表情ボタンがすぐ使える状態になります

> [!NOTE]
> スクリプトは PSD の読み込み自体を行いません（バグ防止のため AE 標準機能に任せています）

### 尺を伸ばしたい場合は「尺を伸ばす（選択）」

伸ばしたいレイヤーを選択して実行すると、そのレイヤーと参照先コンポ・配下をまとめて
指定の長さまで伸ばせます（縮めません／未選択時は何もしません）

### PSDToolKit 命名規則の対応表

| 記法 | PSDToolKit での意味 | EmoLabMaker での扱い |
| --- | --- | --- |
| `*レイヤー名` | 排他表示 | 制御レイヤーで切り替え。兄弟内で排他選択 |
| `レイヤー名` | 任意指定（独立 ON/OFF） | 制御レイヤーで切り替え。チェックボックス |
| `!レイヤー名` | 強制表示 | グレーアウトで操作不可 |
| `レイヤー名:flipx` `:flipy` `:flipxy` | 反転バリエーション（ペア） | 対応ペアを登録し、立ち絵タブの反転チェックボタン（↔左右 / ↕上下）で反転 |

> [!NOTE]
> レイヤー名にカンマ「,」は使えません（リネーム推奨）  
> （マーカーは、表示中レイヤー名の集合をカンマ区切りで保持するため、レイヤー名にカンマがあると壊れます）

<details>

<summary>Tips: セットアップ時の自動処理</summary>

- **コンポ名の一意化**: 複数キャラ設定で壊れないよう、グループコンポを `<ルートコンポ名>_<グループ名>` に改名します
- **重複レイヤー名の一意化**: 同じコンポ内の同名レイヤーに `(2)` `(3)` を付けます
- **デフォルト表情マーカー**: もともとの表示状態を時刻 0 のマーカーとして書き込みます  

</details>

<details>

<summary>Tips: 立ち絵の更新（再セットアップ）について</summary>

`解析してセットアップ / 更新` は何度実行しても安全です（冪等）

- 適用済みのリネーム・制御レイヤー・打ったマーカーはそのまま
- 登録済みレイヤーのエクスプレッションは最新の内容に更新
- 口形状の割り当て設定済みのレイヤーは上書きせず保持
- AE 内で手動追加したレイヤーは差分として登録

</details>

### 設定（タブ下部）

立ち絵タブの表示・挙動を切り替えられます  
設定は Ae 環境に保存され、プロジェクトを跨いで保持されます

| 設定項目 | できること |
| --- | --- |
| **再生ヘッド追従** | 立ち絵タブをアクティブにしたとき、現在時刻の状態に表示を同期 |
| **「!」常時表示レイヤーを表示** | 立ち絵タブに、強制表示の `!` レイヤーを出すか（グレーアウト表示） |
| **非アクティブ階層を隠す** | 上位未選択の階層を、グレーアウトの代わりに非表示にする |
| **インデント幅(px)** | 階層のインデント量。ウィンドウの横幅を狭くしたいときに |


## 2. 表情や衣服を切り替える「立ち絵」タブ

<img width="443" height="305" alt="タイトルなし" src="https://github.com/user-attachments/assets/a4c0eb17-9d0e-483b-b1e1-23a0134e7261" />

鳴花ミコト「なんか動いた」　　鳴花ヒメ「こわ～」

基本的に「制御レイヤー（マーカーでできたコントローラー）」と「不透明度エクスプレッション」で動いています  
PSDToolKit 互換にしたため、排他表示（*）や常時表示（!）に対応しました

> [!NOTE]
> 表情パーツがレイヤー分けされている必要があります  
> PSDToolKit 対応など、ニコニコ動画などにおいてある立ち絵はだいたい多分OKです

> [!NOTE]
> PSDToolKit 非対応の立ち絵、PNG 画像でもセットアップできます  
> 「*」や「!」をあらかじめ先頭につけておくと、立ち絵タブで排他・常時表示が使えます

### 使い方

1. 立ち絵タブで `立ち絵ルート` を選ぶ（立ち絵として設定済みのコンポが一覧に出ます）
2. 目 / 口 / 服 などの階層がインデントで並ぶので、クリックして ON/OFF できます
   - **ラジオ（`*` レイヤー / コンポ）= ラジオボタン**（1 つだけ表示。例: 表情、口の形）
   - **任意指定（無印レイヤー）= チェックボックス**（独立 ON/OFF。例: 頬染め、汗）
   - **常時表示（`!` レイヤー）= チェック済み・グレーアウト**（情報として見えるが操作不可）
3. **上位コンポが選択されていない階層はグレーアウト**します（`*` はコンポにも効きます）

縦に溢れたら右端のスクロールバーで上下、素材が多すぎる場合は階層を折りたたみ推奨です（`▼` マーク）  
マーカーの値が、`おどろき,赤面` のとき、 その階層では おどろき と 赤面 だけが表示されます

<img width="532" height="174" alt="image" src="https://github.com/user-attachments/assets/2faa89bb-db87-4130-bcc9-a7ca15aa080f" />

（削除したいときは、マーカーを右クリック →「マーカーを削除」）

> [!NOTE]
> 再生位置移動で表情が変わった場合は、「更新」ボタンで取り込んでください
> （ScriptUI は再生ヘッド移動を直接検知できないためです）

<details>

<summary>コラム: なぜ不透明度を直接いじらずに、「制御レイヤー」方式を採用しているのか</summary>

立ち絵は、目・口・眉・服…… etc. とパーツが非常に多く、不透明度を 1 枚ずつキーフレームで切り替えるとけっこう大変になりがちです  
そこで「制御レイヤー」に点として情報を書き、各パーツはそれに連動して表示/非表示しています

これによる利点は、

- **後から制御しやすい**
   制御レイヤーの置く場所で立ち絵レイヤーを切り替えられます。レイヤーがキーフレームだらけにならないので、あとから修正しやすいです
- **二重表示が起こりにくい**
   1 つの制御レイヤーを見るため、選択中以外は排他的に隠れ、表情レイヤーが 2 つ表示される、などは起きにくくなります
- **口パク・目パチ・表情を両立できる**
   同じ不透明度をエクスプレッション（式）で書いているため、「しゃべる間だけ口パクで口を上書き」「開き目のときだけまばたき」といった制御ができます

また、レンダリング（エンコード）時間は不透明度方式とほぼ変わらなかったため、利便性の観点からこの方式を採用しています

</details>

<details>

<summary>Tips: 制御レイヤーをどこに置くか（セットアップの `制御`）</summary>

A. 選べるようにしています。立ち絵内部、立ち絵が置かれているコンポジション、どちらでも動きます。使い分けると便利です

- **コンポ1に置く**: 編集中のシーンに制御レイヤーが並ぶので、
  手元でマーカーを調整しやすい（タイムラインで表情を直接いじりたいとき向き）
- **立ち絵コンポの中に置く**: 制御レイヤーが立ち絵と一緒に動くので、
  立ち絵ごとまとまって持ち運べる、見やすい（別シーンに置いても設定が付いてくる）

</details>

### 制御レイヤーの表示/非表示（シャイ）

制御レイヤーは「シャイ（shy）」状態に設定されるので、
タイムライン上部のシャイのトグル（人が手で顔を隠すアイコン）を切り替えると、まとめて隠せます

<img width="154" height="48" alt="Group 141" src="https://github.com/user-attachments/assets/154a25eb-8679-478d-9ab1-a4f104b4b5f1" />

### 立ち絵を一括で切り替える / 表情セット

目・口・眉・衣服など、複数グループの立ち絵状態をまとめて保存し、ワンクリックで切り替えられます

1. 各階層の表情を立ち絵タブで作る
2. 「表情セット」パネルの `保存` → セット名を付ける
3. 切り替えたい位置に再生ヘッドを移動して `適用`
   → 全グループのマーカーが **制御コンポの現在時刻** に一括で書き込まれます

セットは制御コンポ内の `[EmoSet] <名前>` ガイドレイヤーに保存されます（同名で保存すると上書き）  
プロジェクト内の保存なので、使いまわしたい場合は .aep を保存して、別の .aep に取り込んでください  


## 3. 音素データで喋らせる「口パク」タブ

<img width="854" height="282" alt="image" src="https://github.com/user-attachments/assets/152bb63e-869e-4ff0-a86a-319b01f7cd01" />

ソフトウェアトークで、口パクアニメーションをかんたんに作成するための機能です  
音素タイミングファイルを解析して音素レイヤー/ラベルを生成し、リップシンクを行います

※ 音声データの音量の大きさではなく、より制度の高い、音素データ（.lab）で口パクします

> [!WARNING]
> lab ファイルの構造が独自的で特殊だと動かない可能性があります（NEUTRINO なら `full.lab` ではなく `timing.lab` を使ってください）

### 事前準備（.lab ファイルの書き出し方法）

#### A.I.VOICE2

「音素情報ファイルを音声ファイルといっしょに保存する」にチェックを入れてください

<img width="681" height="183" alt="image" src="https://github.com/user-attachments/assets/f847053f-387c-4a46-9ca5-50f3fe2aea7d" />

#### VOICEVOX

[lab ファイルを一緒に書き出す](https://voicevox.hiroshiba.jp/how_to_use/#:~:text=lab%20%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%E6%9B%B8%E3%81%8D%E5%87%BA%E3%81%97) を参照してください

### 使い方（ざっくり）

#### 音素レイヤーの配置

1. 「...」で lab ファイルを読み込む
2. 「音素を選択...」で使用したい音素にチェック
3. セリフの開始位置に移動し、「音素配置」で配置

#### 口パク画像の割り当て

1. 口パク画像の口形レイヤー（あ/い/う/え/お/閉 など）を選択
2. 「口形状の割り当て」パネルで割り当てる
3. 「適用」で各レイヤーに口パク設定を書き込み  
   （場所選択ダイアログが出た場合は、口パクレイヤーを配置する予定のある任意のコンポを選択してください）

音素選択は、どれを選べばよいかわからない場合は、デフォルト（母音+ん）で OK です  
口パクに最低限必要な、基本セット（母音・ん・無音/閉じ系）が選択されます

より細かい口の動きにしたいときは、それぞれのボックスにチェックを入れてください  
多くても閉じ口がわかりやすい s、m、r、w あたりにとどめておくとよさげです

> [!NOTE]
> タイミングは「オフセット(ms)」＝動画先行の原則 が使えます（※範囲外は描画されないので、前後に伸ばしてください）  
> 一部の lab ファイルで発話終了時に口パクが閉じない場合は、「終了に閉じ口」（発話後に口を閉じる）で調整できるかもしれません  
> `<` `>` ボタンは選択レイヤーのマーカーを 1 フレームずつ動かします（発音のズレなど、配置後に微調整したい場合に）

> [!NOTE]
> 現在時刻に有効なレイヤーのうち、一番上にあるものが参照される仕様です。複数ある場合は、「口パクタグ:」機能を使ってください

### 一括で複数の音素データを取り込みする「ファイル一括読み込み」

音素データやテキストを一括で取り込めるようにしています  
複数の音声ファイルがある場合は、lab ファイルを一つずつ選ぶより、音声を配置してからこちらを行うほうが便利です

同じ名前のデータを隣においておくと、複数セリフがあってもすべて秒で設定できるモードです  
音素指定にも同じ「母音+ん / 子音 / すべて」ボタンがありますが、基本的には同じです

> [!NOTE]
> VOICEVOX や アイボスなど、wav ファイル等書き出し時に、同名の lab ファイルも一緒に書き出す機能があります
> 例えば、`セリフ1.wav` と同じ場所に `セリフ1.lab` を置いておくと、一括取り込みできます

### 口パクの準備「口形状の割り当て」

口形（あ / い / う / え / お / ん(閉)）などを、「口形状の割り当て」パネルで、各音素に紐づけできます  
（1 つの口形に複数音素をカンマ区切りで指定可）

> [!NOTE]
> 「使う音素」のうちどの口形にも割り当てられていない音素があると、  
> パネル下部に「⚠ 口形に未割当（閉じ口になります）」と表示されます  
> その音素は閉じ口になるので、必要に応じて対応する口形の行に足してください

> [!TIP]
> 「自動割当」は、選択レイヤー名に「あ/い/う/え/お/ん/閉」が含まれていれば自動で振り分けます  
> 立ち絵のレイヤー名に「あ」などが含まれていれば、全選択 → 自動割当 → 適用 で、手早く一括登録できます

※ 登録ボタンを押すのを、忘れずに行ってください

#### 音素のプリセット例

| 口形 | 音素 |
| --- | --- |
| あ | a |
| い | i |
| う | u, w |
| え | e |
| お | o |
| ん(閉) | N, cl, Q, pau, sil, br<br> **+ どの口形にも属さない音素（子音など）** |

- 子音にも対応させたい場合は、音素行を追加してください  
  音素リストは各行のテキストボックスで自由に編集できます
- 「プリセット」で初期値に復元できます（初期値は上表のとおり）

### 音素について

このスクリプトは特定の音素リストでフィルターはしていないため、実質どんな音素にも対応しています

| 種類 | 音素例 |
| --- | --- |
| **母音** | a, i, u, e, o |
| **特殊** | N (ん), cl (っ), pau (ポーズ), sil (無音) 、br (ブレス) |
| **子音** | k, s, t, n, h, m, y, r, w, g, z, d, b, p など |
| 日本語など | あ, か, さ, た, な, ー、開、閉、笑、困、△ ……？ |

### lab ファイルってなに？

<img width="970" height="158" alt="Frame 110" src="https://github.com/user-attachments/assets/229e08b5-f579-4dcd-a347-27d9f52edcb2" />

lab ファイルは以下のように 3 つに区分けされた形式のテキストファイルです

```
0 1000000 sil
1000000 2500000 k
2500000 5000000 o
5000000 7500000 N
7500000 10000000 n
10000000 12500000 i
12500000 15000000 ch
15000000 17500000 i
17500000 20000000 w
20000000 22500000 a
22500000 30000000 sil
```

- 1 列目 → 開始時間（100 ナノ秒単位）
- 2 列目 → 終了時間（100 ナノ秒単位）
- 3 列目 → 音素

### 対応ソフトウェア

上のような、単純な形式の音素ファイルなら対応しています

## 4. 自動まばたきを行う「目パチ」タブ

<img width="440" height="219" alt="image" src="https://github.com/user-attachments/assets/a0103a16-54d5-4247-ade5-0f7603f587ec" />

マーカー不要の自動まばたきを設定できます

1. 開き目レイヤーを選択 → `開き目` の「割当」
2. 閉じ目レイヤーを選択 → `閉じ目` の「割当」
3. 任意で、ジト目などのレイヤーを選択 → `中間` を「割当」
4. 間隔・速度・ランダム% を調合して `目パチ設定`

「解除」で表情切替のみの状態（未登録だった場合は不透明度 100）に戻ります

> [!NOTE]
> 「セットアップ」済みに更に設定すると、現在の表情が「開き目」のときだけまばたきします  


## 制限・制約

- **PSD 側の構造変更（レイヤー追加など）は既存コンポに反映されません**  
   フッテージ再読み込みはピクセルのみ更新します  
   構造が変わったら再インポート＋再セットアップが必要で、打ったマーカーは旧制御レイヤーに残ります
- **セットアップ後にコンポを改名しないでください**  
   エクスプレッションの `comp("名前")` 参照と `[Emo] 名前` の対応が切れます
- **同じ立ち絵で2つの〈制御〉を設定しないでください**
   一度作成した〈制御〉の場所以外にもう一つの〈制御〉を作成すると、参照先が 2 つになって壊れます
- 表情切替は、レイヤーグループ（目・眉・口など）ごとに対象コンポを選んでマーカーを打つ方式です  
   複数グループをまとめて切り替える場合は、立ち絵タブの「表情セット」を使ってください
- 〈ルート〉と〈制御〉が違う場合で、立ち絵のコンポを動かすと、シーン自体とズレたように見えます
   開始位置（先頭）を合わせてください（立ち絵レイヤーは左右に動かさないでください）  


## トラブルシューティング

### 立ち絵タブ: 表情が切り替わらない

- セットアップタブで自動セットアップ済みか（制御コンポ内に `[Emo] <対象コンポ名>` があるか）確認してください
- 立ち絵ルートに、セットアップ済みのコンポを選んでいるか確認してください

### 口パクタブ: 口パクが反映されない

口形状の割り当てで各口形に音素が割り当てられているか確認してください  
`[Lab]` レイヤーがあるコンポを「適用」のダイアログで正しく選んでいるか確認してください

### 口パクタブ: lab ファイルが読み込めない

ファイルの文字コードが UTF-8 または Shift-JIS であることを確認してください

### 音ズレが起きた

指定したコンポジションの開始位置と、立ち絵の開始位置（先頭）をあわせてください
