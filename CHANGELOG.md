# CHANGELOG

### v2.0.1

- **立ち絵タブの `*` 排他判定のバグを修正**: `*閉_にゃ` `*閉_む` のように
  「`*` + アンダースコアを含む名前」が多数あると、`*閉_` がキャラ prefix として
  誤検出・除去され、**排他マーカー `*` ごと消えてラジオが任意指定（チェックボックス）に
  化けて**いました。キャラ prefix 検出（`detectDominantPrefix`）が `*` / `!` マーカーを
  含む候補を prefix にしないよう修正。これで同一グループ内の `*` レイヤーはすべて正しく
  ラジオ（排他）になります（`*閉_にゃ` は `にゃ` と区別して `閉_にゃ` と表示）。

### v2.0.0（破壊的変更）

PSD 互換化（自動セットアップ・立ち絵タブ・口形状マッピング）の整備により実質的に重複していた
レガシー機能を撤去し、構成を整理しました。

- **「詳細」タブ（旧レイヤー選択）を廃止**: 単一コンポ・ラジオ単一選択の手動グリッドは、
  立ち絵タブ（集合モデルでラジオ＋チェックボックス＋階層を統一表示）とセットアップタブの
  自動登録で完全に代替できるため削除。表情セット（`[EmoSet]`）は立ち絵タブに引き続き存在します
- **口パクの「名前マッチ」式（レガシー）を廃止**: レイヤー名＝音素名で判定する旧方式を削除し、
  PSDToolKit 互換の **口形状マッピング**（音素 CSV 埋め込み・emo フォールバック）に一本化
- タブ構成は **セットアップ → 立ち絵 → 口パク → 目パチ** の 4 つに（初期選択は「立ち絵」）
- 上記に伴い未使用となった内部関数（`registerSelectedLayers` / `unregisterSelectedLayers` /
  `collectMarkerNames` / `countRegisteredLayers` / `writeMarkerName` / グリッド計算ヘルパー等）を整理
- マーカーデータ形式（`[Emo]` / `[EmoSet]` / 式）は不変。既存プロジェクトはそのまま開けます

### v1.22.0

- タブの並びを **セットアップ → 立ち絵 → 口パク → 目パチ → 詳細** に変更（作業フロー順）。初期選択は従来どおり日常のハブである「立ち絵」
- 「PSD」タブを **「セットアップ」** に改称（ファイル形式名より「最初にここで準備する」役割が伝わるように）。ヘルプ・helpTip 等の文言も追随
- スクロール枠がウィンドウのリサイズ/移動で壊れる不具合を安定化。パネル自身の size を読んで設定するとフィードバックで値が壊れていくため、**幅・高さともウィンドウサイズから他要素を引いて算出**する（読み取りに依存しない）。立ち絵タブは「ウィンドウ高さ − 上下の要素」で可変高さ、口パクタブは固定高さ + スクロールバー。幅もウィンドウ由来にしてスクロールバーが画面外へ消えないようにした。マウスホイールは AE の ScriptUI が非対応のためスクロールバーで操作する

- 外側コンポをルートに選び、それが立ち絵フォルダ 1 つを包むだけのとき、包まれた立ち絵ノード（実質ルート）にはチェックボックスを出さない（立ち絵全体のトグルは不要）
- 口形状マッピングの「自動割当」を **1 行（口形）= 1 口パクレイヤー** に変更。1 つの口形に複数の口パクレイヤーが一致した場合は、使う 1 枚を選ぶダイアログを表示（従来は `*いー,*母_い` のように複数割り当てて口が二重表示になることがあった）。1 レイヤーは 1 行までしか使わない
- 立ち絵ツリーで、サブ階層を持つフォルダは**ヘッダ行自体をそのフォルダの操作（`▼☑その他` / `▼◉…`）に集約**するよう変更。従来は親側の選択肢とサブ階層ヘッダで二重表示になっていたのを解消。ルートが立ち絵フォルダを包むだけのとき（外側コンポをルートに選んだ場合）は「（ルート）」行を省略
- 立ち絵タブ・口パクタブの可変高さスクロール枠が、**スクリプトウィンドウ自体をリサイズすると壊れる**不具合を修正。リサイズ時はツリー/行を作り直さず、中身を測り直してスクロールバーを再フィットするだけにし、パネルをコンテンツ高さへ伸縮させる再レイアウトを避けた
- 立ち絵を外側コンポに入れて**そちらをルートに選んだ**場合でも、各コンポ自身のレイヤー名から prefix を検出して `*`/`!` の種別とラベルを正しく出すように修正（従来はルート名の prefix しか剥がせず、外側コンポ選択時に `T_Mhime_!腕` のように冗長表示＆`!` 未検出になっていた）。`母_お` のように base 名に `_` を含む名前も保持
- 上位（ルートや各階層）に置かれた**フォルダ**も種別どおりの操作対象として表示するように修正。`*` フォルダ＝ラジオ（従来通り）、無印フォルダ＝チェックボックス（丸ごと表示/非表示）、`!` フォルダ＝強制表示（グレー表示）。従来は `*` 以外のフォルダがコンテナ扱いで操作できなかった。※反映には再セットアップが必要
- 立ち絵ルート直下に置かれた `*` レイヤー（立ち絵全体の衣装切替など）もラジオ対象として表示。立ち絵名と紛らわしくないよう「（ルート）」と表示する
- 制御ヌル（`[Emo]`）が複数あるとき逆順に並んでいたのを作成順に修正（位置は従来通り最上位のまま）
- **重要バグ修正**: ネストした三項演算子が ExtendScript で誤評価され、(1) 立ち絵タブの「左右反転」が上下反転になる、(2) PSDタブの「* 排他」ボタンが `!` を付与する、という不具合が発生していたのを if/else に書き換えて修正。目パチ式の rolePhase も同様に予防的に修正
- 立ち絵が複数あるときの口パク競合対策として「口パクタグ」を追加。口形状マッピングでタグ（例: lab ファイル名の prefix `zunda_`）を指定すると、その文字列を名前に含む `[Lab]` レイヤーだけに反応する（空＝従来通り最初の `[Lab]`）
- 音素⇔口形の割り当ては「現在を取込」→編集→「適用」で **lab を再配置せず** 後から変更可能（マーカーは置いたまま不透明度式だけ更新）。「現在を取込」は口パクタグも復元
- ポーズを個別の `*` フォルダで包む立ち絵（例 `奥腕 → *ひみつ(フォルダ) → ひみつ(絵)`）で、フォルダを 1 つのラジオ選択肢に集約。中身が「絵」だけの `*` フォルダは冗長なサブ階層を出さず、フォルダ選択時に中身を常時表示（中身にさらに `*` がある本物の階層はこれまで通り展開）。※反映には PSD タブで再セットアップが必要
- 口形状マッピングの行領域を縦スクロール対応に（口形を増やしても隠れず、追加/適用ボタンは常に下に残る）
- ファイル一括読み込みに「使用する音素」の絞り込みを追加（既定は母音＋ん＋閉じ系。空＝すべて。「母音+ん」「すべて」ボタン）
- PSD タブに命名ショートカットを追加（選択レイヤーに `*`（排他/ラジオ）/`!`（強制表示）を付与・除去）
- 目パチの解除をコンポジショングループ単位に変更（開きだけ解除されて中間/閉じが残る不整合を防止。「解除(コンポ)」「解除(一覧)」ともコンポ単位で一括）
- 右下にバージョン番号を表示
- 立ち絵ツリーの表示名がまだ冗長だった不具合を修正（支配的 prefix を多数決で検出し、`<立ち絵>_!腕` → `腕` のように剥がす。prefix を持たないコンポや複数キャラ混在にも対応）
- 立ち絵ツリーで、子コンポを持たない末端グループ（目/口/服 など）も「コンポジションごとに」折りたためるよう改善（折りたたみ三角が出ないケースを解消）
- 立ち絵ツリーをマウスホイールでスクロール可能に（AE の ScriptUI 対応状況に依存。非対応環境はスクロールバー操作にフォールバック）
- 口形状マッピングの「プリセット」を、初期状態（あ/い/う/え/お/ん）への確実なリセットに変更
- 口形状マッピングに「現在を取込」を追加（既存の口パク設定済みレイヤーを読み取って各行へ反映）
- 自動割当で候補が複数ある口形レイヤーは、割当先を選べるダイアログを表示

### v1.19.2

- **目パチの「重なり」を式レベルで防止**: 開き目表情が選択中（blinkEnabled）のときは、マーカー集合の membership を見ずに**位相だけで表示を1枚に確定**（開き=phase0 / 中間=phase1 / 閉じ=phase2）。これでマーカーに他の目名が混ざっていても、開き/中間/閉じが決して同時表示されない

### v1.19.1

- **目パチの開き/中間/閉じが同期しない不具合を修正**: まばたき時刻を `seedRandom`/`random` で算出していたが、環境によってレイヤーごとに乱数が揃わず、開き/中間/閉じの瞬きが別タイミングになって隙間（目消え）や重なり（複数表示）が発生していた。乱数を **`n`（サイクル番号）だけの純粋な数式ハッシュ**に置き換え、全レイヤーで必ず同一の瞬きスケジュールを共有するようにした（完全同期）
  - ※ 反映には目パチ設定を再適用してください（開き/中間/閉じを揃えて1回で適用）

### v1.19.0

- **ファイル一括読み込み（wav/txt/lab）を追加**: 口パクタブに「ファイル一括読み込み」パネルを新設。配置対象（口パク(lab) / テキスト(txt) / 音声(wav)）は設定でオンオフでき、永続化される
  - **A方式「ファイルを選択して配置」**: wav/txt/lab をまとめて選択 → 同名どうしを1組として現在のコンポに配置（AE はパネルへのドラッグ&ドロップ非対応のためファイルピッカー）。音声は取り込み＆配置、lab は口パクマーカー、txt は字幕テキストレイヤー
  - **B方式「選択音声の隣を取込」**: タイムラインで音声レイヤーを選択 → そのソースと同じ名前の `.lab`/`.txt` が隣にあれば取り込む（lab はその音声レイヤーに配置）
  - テキストは UTF-8 の txt をそのまま字幕レイヤー化する簡易版（音声に紐づけた高度な連携は今後対応）

### v1.18.2

- **目パチで目が消える不具合を修正**: 立ち絵の目グループのマーカーが「何も選択されていない空集合(`""`)」のとき、目パチ式が空文字を通常分岐で処理して全部の目を非表示にしていた（⚠未選択の階層で発生）。`null`（マーカー皆無）と同様に**空のときも単独まばたきにフォールバック**し、目が消えないようにした
  - ※ 修正を反映するには目パチタブで「目パチ設定」を再実行してください

### v1.18.1

- **口パク「適用」が開いているコンポに依存しない不具合を修正（二重表示の真因）**: グループ優先の抑制処理が `getActiveComp()`（今開いているコンポ）を走査していたため、**口レイヤーとは別のコンポを開いて「適用」すると、無関係なコンポのレイヤー（目など）に口パク合成式を付けてしまい**二重表示になっていた。**割り当て済みレイヤーが実際に入っているコンポ（`containingComp`）**だけを対象にするよう修正
- **音素レイヤー [Lab] の配置を必須にしない**: 合成式は実行時に [Lab] を探すため、未配置でも「適用」を中止せず、配置予定のコンポを選んで進められるようにした（[Lab] が1つあれば自動採用、複数なら選択）
  - ※ 既に誤ったコンポのレイヤー（目など）に口パク式が付いてしまっている場合は、そのレイヤーを選んで口パクタブの「解除」で表情切替に戻してください

### v1.18.0

- **口パク/目パチの音素ソースを自動検出（間違ったコンポへのセットアップを防止）**: 「口パク適用」「口パク設定」時に音素ソースを手で選ばせる代わりに、**`[Lab]` が実在するコンポを自動特定**する。0個なら警告（先に「音素配置」が必要）、1個なら自動採用、複数なら `[Lab]` を含むコンポだけから選択。
  - 従来は音素ソースに `[Lab]` の無い間違ったコンポを指定でき、口/目が誤った発話判定で二重表示される原因になっていた

### v1.17.0

### v1.16.0

  - 頭/尾トリムの終端も「ラボ終了時刻」基準に統一（より正確に末尾をカット）

### v1.15.2

- **PSDタブのルート/制御ドロップダウンを「PSD立ち絵ルート候補」に変更**: v1.13.0の「現在開いているコンポのみ」だと、作業用コンポを開いていると肝心の立ち絵コンポが選べなかった。**プロジェクト直下（ルート階層）のコンポのうち、PSD由来（`<名前> レイヤー`/`Layers` フォルダを持つ、または .psd フッテージを含む）かセットアップ済み（[Emo]あり）のもの**だけを表示するように変更。ネストされた中間コンポ（反転バリエーション等）や無関係な作業コンポは出ない。検出できない場合は直下コンポ全体→アクティブコンポにフォールバック

### v1.15.1

### v1.15.0

- **再セットアップで口パク/目パチが壊れる問題を修正（主因A）**: 同名コンポ修正(v1.13.0)でコンポがリネームされると、保持していた口パク/目パチの合成式が**古いコンポ名を参照したまま**になり、目が消える/立ち絵と被る等が起きていた。リネーム時に**制御レイヤー名（マーカー保持のまま）と式内の旧コンポ名参照を新名へ自動移行**するようにした（`migrateGroupRename`）。再適用不要に
- **発話中に立ち絵の口が二重表示される問題を修正（主因B・口パク優先）**: 口パク適用時、同じコンポ内の「他の表情登録済み口レイヤー（休め口など）」も**発話中は非表示**にする（無音中は表情に従う）。これで「口パクの口」と「立ち絵で選んだ口」の二重表示を防止。結果ダイアログに対象枚数を表示
  - 目パチの「目が消える/被る」は主因A（参照壊れ）が主因のため、上記の自動移行で解消（まばたきは従来どおり開き目表情のときだけ作動）

### v1.14.0

- **口パクのタイミング設定を拡張＋永続化**: 「オフセット(ms) / 頭(ms) / 尾(ms)」の3つにし、すべて `app.settings` に永続化（次回起動時も保持）
  - **オフセット**: 全マーカーを一律シフト（映像先行）。既定を **-67 → 0** に変更
  - **頭(ms)**: 口パク開始の前マージン。この時間ぶん開始を遅らせ、それまでは口を閉じ/立ち絵に戻す
  - **尾(ms)**: 口パク終了の後トリム。この時間ぶん終了を早め、以降は口を閉じ/立ち絵に戻す
  - 実装は **[Lab] レイヤーの in/out を頭/尾ぶん詰める**方式（`findPhonemeLayer` が範囲外を無視するので自然にゲートされる）。**音声・映像レイヤーに付けた [Lab] には音を切らないよう未適用**（ヌル [Lab] レイヤーで使用。配置時に通知）
